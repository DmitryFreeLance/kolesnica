# MAX-бот «Колесница»

Бот для мессенджера MAX на `Java + SQLite + Maven + Docker`.

## Что реализовано по ТЗ
- 6 ключевых сценариев:
  - запись на сезонную переобувку
  - поиск ближайшего филиала
  - расчет цены (в формате «от ... ₽»)
  - хранение шин
  - отзыв / жалоба
  - связь с оператором
- Кнопочная логика (inline keyboard), все кнопки с эмодзи.
- Во всех шагах доступны:
  - `⬅️ Назад`
  - `🏠 Главное меню`
  - `👩‍💼 Оператор`
- Справочник филиалов в SQLite как единый источник данных.
- Фильтрация филиалов по городу, району и режиму работы.
- Функция ближайшего филиала по геолокации (расчёт по координатам).
- Сохранение данных клиента и заявок в SQLite (`requests`).
- Контроль незавершенных диалогов через таблицу `user_sessions`.
- Автоклассификация входящего текста в один из 6 сценариев.
- Кнопка `👩‍💼 Оператор` сразу отправляет номер оператора.
- Админ-панель в боте:
  - изменение телефона оператора
  - добавление/удаление админов
  - добавление/удаление филиалов

## Переменные окружения
Скопируйте `.env.example` и заполните:

- `BOT_TOKEN` (обязательно)
- `MAX_API_BASE_URL` (по умолчанию `https://platform-api.max.ru`)
- `BOT_POLL_TIMEOUT` (по умолчанию `1`, максимально частый polling)
- `BOT_POLL_LIMIT` (по умолчанию `1000`)
- `BOT_DB_PATH` (по умолчанию `./data/bot.db`)

## Локальный запуск
```bash
mvn -DskipTests package
BOT_TOKEN=... java -jar target/max-bot-kolesnica-1.0.0.jar
```

## Админ-панель
- Откройте чат с ботом и отправьте `/admin`.
- Первый пользователь, вызвавший `/admin`, автоматически становится админом.
- Дальше управление делается кнопками в админ-панели.

## Запуск в Docker
```bash
docker build -t kolesnica-max-bot .
docker run --rm \
  -e BOT_TOKEN=... \
  -e BOT_DB_PATH=/opt/bot/data/bot.db \
  -v $(pwd)/data:/opt/bot/data \
  kolesnica-max-bot
```

## Где менять справочник филиалов
Данные филиалов лежат в SQLite-таблице `branches`.
Обновление адресов/телефонов/графиков можно делать SQL-запросами без изменения Java-кода.

## Использованные API MAX
- `GET /updates` (long polling)
- `POST /messages`
- `POST /answers`
- `GET /me`

Основано на документации MAX для разработчиков: [dev.max.ru/docs-api](https://dev.max.ru/docs-api)
